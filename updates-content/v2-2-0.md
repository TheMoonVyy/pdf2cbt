### Enhanced Test Results Page and Fixed Bugs in Test Maker

#### Added Pagination, Sorting, and Detailed Test Cards in My Tests section of Test Results Page

**Pagination**: Instead of loading and displaying all test cards in **My Tests**, cards are now paginated (Page 1, 2, 3, ...) with 20 cards per page for easier navigation.

**Sort Tests**: You can now sort tests in **My Tests** using the following options:
1. Test Added (when the test was added to your database, either after attempting or importing it)
2. Test Start Date & Time
3. Test End Date & Time
4. Test Name (sorted lexicographically, similar to how file explorers sort filenames)
5. Accuracy
6. Number of Questions Answered Correctly
6. Number of Questions Answered Incorrectly
8. Average Time Spent per Question
9. Total Time Spent
10. Marks Percent (i.e., (marks obtained / max marks) * 100)
11. Marks Obtained
12. Max Marks
13. Percent of Questions Attempted (i.e., (questions attempted / total questions) * 100)
14. Number of Questions Attempted
15. Total Number of Questions

Sorting options from 5 to 15 can be further divided by question type: **All**, **MCQ**, **MSQ**, **NAT**, and **MSM**.

An **ascending/descending toggle button** is provided (located right to the **Sort By** button).

Before this feature (versions prior to v2.2.0), tests were sorted by **Test Added** in descending order. This remains the default unless changed.

**Detailed Test Cards**: Due to the addition of sorting, more metrics are now available. These can be viewed on test cards by toggling the **Show/Hide more metrics on cards** button (icon located on left to the **Sort By** button).

---

#### Maximum Marks Display and Keyboard Navigation in Questions Panel of Test Results Page

In the Questions Panel:
- Maximum marks are now displayed in the format **Marks Obtained / Max Marks**. Previously, only Marks Obtained was shown.
- Questions can now be navigated using the keyboard left and right arrow keys to move to the previous and next question, respectively.

---

#### Bug Fixes in Test Maker

The following issues were fixed/changed in the PDF Cropper:

1. Previously, when switching from **Line Cropper** mode to **Edit** mode and back to **Crop** mode, the drawn lines were cleared. They are now retained.
2. In **Edit** mode, copying and pasting a region/image previously auto-incremented the question number by 1. This behavior has been removed. Now, only **Crop** mode auto-increments question numbers when a region is cropped.
